구조 변화

운영 보드와 창작 작업 뷰를 분리해 혼용을 막는 방향

이전 혼선

  • CLE2-7 STUDIO와 CLE3 창작 시스템 경계가 문서상 혼용
  • 품질 관리가 단일 흐름으로 뭉쳐 있음
  • three-body-comic 실제 파일 구조와 CLE2 문서 구조가 느슨함

현재 정리

  • STUDIO = 운영 큐 / 담당 / 상태 추적
  • Workspace = 회차 목적 / 장면 / 패널 / 검수 / 결과
  • Lint = 규칙 위반, Vision QA = 전달력/승인 판단
