# GOAL — 결과물 쇼케이스 & 피드백 루프

## 목표
각 요구사항에 대해 결과물(Deliverable)을 버전별로 등록하고, 팀원이 피드백을 남기면 재작업으로 이어지는 폐루프를 웹앱에 통합한다. 이전 버전은 보존되어 히스토리를 추적할 수 있다.

## 성공 기준 (Definition of Done)
- [ ] 요구사항 상세 페이지에 결과물 섹션이 표시된다
- [ ] 결과물 등록 시 버전 번호가 자동 증가하며 이전 버전이 보존된다
- [ ] 각 결과물 버전에 피드백(텍스트)을 남길 수 있다
- [ ] 피드백을 바탕으로 새 버전 결과물을 등록할 수 있다
- [ ] 결과물에 외부 링크(GitHub, 이미지 등)를 첨부할 수 있다
- [ ] GitHub Pages 공유 링크가 생성된다
- [ ] UI가 기존 구조에서 자연스럽게 통합되어 복잡하지 않다

## 범위
### 포함 (In Scope)
- 결과물 버전 관리 (localStorage 기반)
- 피드백 등록 및 표시
- 외부 링크 첨부 (URL 다중)
- GitHub Pages 공유 링크 복사
- 요구사항 상세 페이지 통합

### 제외 (Out of Scope)
- 파일 업로드 (이미지 바이너리 등)
- 백엔드 서버 저장
- 실시간 웹소켓 피드백
- 알림 이메일 발송
