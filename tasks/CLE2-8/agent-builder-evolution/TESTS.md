# TESTS — agent_builder_public, CLE2 구조 흡수하여 커뮤니티 중심 레포로 진화

## 검증 기준

### 기능 테스트
| # | 테스트 항목 | 방법 | 기대 결과 | 통과 |
|---|------------|------|-----------|------|
| 1 | tasks/ 폴더 구조 | 리포 확인 | tasks/ + _template/ (GOAL/PLAN/STATUS/TESTS) 가 존재한다 | ☐ |
| 2 | AB-N 네이밍 규칙 | 문서 리뷰 | 스터디 미션 → 태스크 매핑 규칙이 문서화되어 있다 | ☐ |
| 3 | team-memory 구조 | 리포 확인 | context/records/ + context/registry/ 디렉토리가 존재한다 | ☐ |
| 4 | memory-ingest 연동 | 실행 테스트 | 스터디 기록이 memory record로 저장 가능하다 | ☐ |
| 5 | SPA 대시보드 홈 | 브라우저 확인 | 스터디 현황(전체/진행중/완료/진행률)이 표시된다 | ☐ |
| 6 | SPA 미션 페이지 | 브라우저 확인 | 주차별 미션 리스트 + 필터/검색이 동작한다 | ☐ |
| 7 | SPA 멤버 페이지 | 브라우저 확인 | 팀원별 제출 현황 + 역할이 표시된다 | ☐ |
| 8 | SPA 위키 페이지 | 브라우저 확인 | 스터디 노트가 위키로 정리되어 표시된다 | ☐ |
| 9 | 이슈 템플릿 | 리포 확인 | .github/ISSUE_TEMPLATE/에 미션 제출용, 개선 제안용 템플릿이 있다 | ☐ |
| 10 | Discord 알림 | 통합 테스트 | 미션 제출 시 Discord에 알림이 전송된다 | ☐ |
| 11 | README living example | 문서 리뷰 | README가 이 레포의 CLE2-style 운영 방식을 설명한다 | ☐ |
