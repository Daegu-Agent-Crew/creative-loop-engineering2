# PLAN — 매일 CLE2 개선 질문하기

## 실행 계획

### Phase 1: 질문 생성 로직 설계
- **담당**: 대구루
- **입력**: CLE2 전체 태스크 상태 (tasks/*/STATUS.md), 4가지 패턴 정의
- **출력**: 4가지 패턴 × 태스크 매트릭스에서 랜덤 질문 선택 로직
- **예상 시간**: 1회 설계

### Phase 2: 일일 실행 (매일 아침)
- **담당**: 대구루 (cron 또는 heartbeat 기반)
- **입력**: 전일 CLE2 상태, 랜덤 패턴 선택
- **출력**: 개선 질문 + 생각 정리 → 이슈 #22 코멘트 등록
- **예상 시간**: 매일 약 5-10분

### Phase 3: 월간 리뷰
- **담당**: 대구루
- **입력**: 한 달간 누적된 질문/답변
- **출력**: CLE2 개선 로드맵 업데이트 제안
- **예상 시간**: 월 1회

## 4가지 패턴 질문 템플릿

### 1. 축적 구조 (일회성 → 복합 시스템)
- "어제 풀었던 문제의 해법을 재사용 가능한 형태로 저장했나?"
- "오늘 작업이 100번째에도 처음부터 시작할까, 라이브러리에 쌓일까?"

### 2. 멀티 에이전트/멀티 관점 오케스트레이션
- "이 문제를 한 가지 관점으로만 보고 있지 않은가?"
- "하위 작업으로 쪼개 병렬로 위임할 수 있는가?"

### 3. 검증/재현성
- "오늘 결과물을 누가 다시 검증할 수 있는 형태로 남겼나?"
- "확신하는 것 중 실제로 재점검하지 않은 게 있나?"

### 4. 프로세스 재설계로 시간 단축
- "이 작업의 병목은 무엇인가? 어떤 단계를 통째로 없앨 수 있는가?"

## 의존성
- CLE2 이슈 #22 접근 권한
- tasks/*/STATUS.md 최신 유지

## 리스크
- 질문이 형식적으로 흐를 위험 → 구체적인 태스크 상태와 연계하여 실질적 질문 도출
