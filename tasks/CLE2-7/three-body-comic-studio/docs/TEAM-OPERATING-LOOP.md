# TEAM OPERATING LOOP — Three Body Comic Studio

## 역할

### sfex11
- 작품 방향 최종 판단
- 파일럿 Go/Pivot/Stop 게이트 결정

### 대구루
- 시스템 문서화
- 태스크 구조 관리
- 이미지 생성 실행 표준화
- PR 리뷰 흐름 정리

### 레노버
- 가설 검증 구조 설계
- results 해석 지원
- 실험 설계 보조

### 이미지/대본 담당자
- 실제 에피소드 제작과 수정 반영
- 패널 승인 전 품질 피드백 제공

## 주간 루프
1. 이번 주 대상 에피소드와 가설 선택
2. 대본/콘티 준비 상태 확인
3. 패널 생성과 리뷰 라운드 실행
4. 배포 준비와 결과 기록 슬롯 생성
5. 배포 후 관측 데이터 수집
6. 다음 라운드 변경점 확정

## PR 리뷰 루프
- 문서 변경: CLE2 또는 three-body-comic에 PR 생성
- 패널 후보: 링크 또는 스크린샷과 함께 deliverable 등록
- 리뷰 코멘트: 선택/수정/보류로 분류
- 머지 후: 관련 task 상태와 results 반영

## Discord 알림 기준

### 알릴 이벤트
- 에피소드 단계 변경
- 패널 승인 완료
- 결과 기록 시작
- 리뷰 요청 생성

### 알리지 않을 이벤트
- 단순 오타 수정
- 내부 초안 저장
- 자동 재시도 중간 로그

## handoff 규칙
- 대구루 -> 이미지 담당자:
  - 패널 목적, 스타일 팩, 실패 로그가 준비되어 있어야 함
- 이미지 담당자 -> 리뷰어:
  - 후보 2개 이상 또는 단일 확정안과 선택 이유가 있어야 함
- 리뷰어 -> 레노버:
  - 배포 버전과 관측 포인트가 정리되어 있어야 함

## 위험 신호
- 패널 실패가 누적되는데 이유 분류가 없음
- 스타일 편차를 감으로만 말하고 기준이 없음
- 가설 검증 없이 조회수만 보고 결론을 냄
- 한 명이 에피소드 전체를 끌고 가며 병렬 작업이 사라짐
