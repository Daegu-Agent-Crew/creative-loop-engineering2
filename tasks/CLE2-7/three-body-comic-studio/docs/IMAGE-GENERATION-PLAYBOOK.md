# IMAGE GENERATION PLAYBOOK — OpenClaw(GLM) x Codex CLI

## 문제 정의
현재 병목은 "이미지가 안 나온다" 자체보다, OpenClaw(GLM) 측이 Codex CLI의 이미지 생성 가능 여부와 호출 방식을 일관되게 이해하지 못하는 데 있다. 따라서 우선순위는 모델 교체가 아니라 실행 규약의 표준화다.

## 원칙
- 이미지 생성 요청은 항상 구조화된 입력으로 만든다.
- 한 번에 모든 걸 해결하려 하지 말고, 패널 단위로 시도한다.
- 실패 원인을 `도구 호출 실패`, `프롬프트 품질`, `스타일 불일치`로 나눠 기록한다.

## 표준 입력 조립 순서
1. 장면 요약
2. 패널 목적
3. 등장 인물과 감정
4. 구도/카메라 지시
5. 스타일 팩
6. 금지 요소와 제약조건

## 표준 프롬프트 템플릿

```text
[장면 요약]
{scene_summary}

[패널 목적]
{panel_goal}

[캐릭터]
{characters}

[구도]
{camera_direction}

[스타일]
{style_pack}

[제약조건]
{constraints}
```

## OpenClaw 에이전트 실행 규칙
- "Codex는 이미지 생성 못 함"이라고 가정하지 않는다.
- 이미지 생성 요청 시 반드시 `패널 ID`, `스타일 팩`, `목표 출력`을 먼저 정리한다.
- 한 번 실패하면 즉시 포기하지 않고 아래 재시도 순서를 따른다.

## 재시도 순서
1. 동일 프롬프트로 도구 오류 여부 재확인
2. 장면 설명 길이 축소
3. 스타일 문구를 더 명확한 레퍼런스로 교체
4. 캐릭터/배경/구도를 분리해 프롬프트 단순화
5. A/B 후보 2개 생성 후 선택

## 실패 분류

| 코드 | 의미 | 대응 |
|------|------|------|
| `tool-call` | 도구 호출 자체 실패 | 호출 방식 점검, 재시도 |
| `style-drift` | 기존 화풍과 이질적 | 스타일 팩 강화, 레퍼런스 명시 |
| `composition` | 구도 불일치 | 카메라 지시 구체화 |
| `character-drift` | 인물 외형 흔들림 | 캐릭터 규칙 재주입 |
| `low-signal` | 장면 목적이 흐림 | 패널 목적 문장 재작성 |

## A/B 비교 규칙
- 최소 2개 후보를 동일 패널 ID에 묶는다.
- 선택 이유를 한 줄로 남긴다.
- 선택 기준: 감정 전달, 서사 명확성, 화풍 일관성, 수정 비용

## 추천 기록 포맷

```md
# ATTEMPT LOG — EP001-S02-P03

- attempt: 02
- result: style-drift
- summary: 표정은 좋아졌지만 배경 질감이 다른 패널들과 이질적
- next-action: watercolor-v3 스타일 규칙 강화 후 재시도
```

## 후속 개발 포인트
- 프롬프트 템플릿을 문자열이 아니라 구조화 데이터에서 조립
- 시도 로그를 CLE2 또는 three-body-comic UI에서 비교 가능하게 표시
