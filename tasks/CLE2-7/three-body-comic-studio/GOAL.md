# GOAL — 삼체 연재 만화 프로젝트 전용 관리 시스템

## 목표
삼체 연재 만화 프로젝트의 에피소드 제작, 패널 이미지 생성, 리뷰, 가설 검증 흐름을 하나의 운영 체계로 묶는 전용 관리 시스템을 설계한다. CLE1의 5-Phase 루프와 CLE2의 GOAL/PLAN/STATUS/TESTS 체계를 결합해 팀 병렬 작업과 품질 관리를 동시에 다룬다.

## 성공 기준 (Definition of Done)
- [ ] 에피소드 단위 파이프라인과 패널 단위 상태 모델이 문서화되어 있다
- [ ] OpenClaw(GLM)에서 Codex CLI 이미지 생성을 호출하는 표준 절차가 정리되어 있다
- [ ] 이미지 생성 실패, 스타일 편차, A/B 선택을 다루는 품질 관리 루프가 정의되어 있다
- [ ] 가설 검증용 `results.md` 템플릿과 상태 추적 규칙이 준비되어 있다
- [ ] CLE2 대시보드와 tasks 문서에서 CLE2-7 진행 상태를 추적할 수 있다

## 범위
### 포함 (In Scope)
- 삼체 프로젝트 전용 에피소드 제작 단계 정의
- 패널 상태 모델 정의
- 이미지 생성 실행 매뉴얼 및 프롬프트 조립 규칙
- 가설 실험 기록 구조와 results 템플릿
- 팀원 및 에이전트 병렬 작업 분담 구조
- CLE2-7 task 문서와 CLE2 대시보드 반영

### 제외 (Out of Scope)
- 이미지 생성 모델 자체 개발
- GA4 운영 계정 실제 연결 완료
- Discord 인프라 운영 자동화 완료
- three-body-comic 저장소 전체 구현 완료
