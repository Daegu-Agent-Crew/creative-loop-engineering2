# PLAN — 삼체 연재 만화 프로젝트 전용 관리 시스템

## 실행 계획

### Phase 1: 요구사항 정제 및 시스템 경계 확정
- **담당**: 대구루
- **입력**: Issue #17, 이전 대화 문서, three-body-comic PRD v2.1
- **출력**: 시스템 목표, 범위, 핵심 객체 정의
- **세부**:
  - 삼체 프로젝트에서 CLE2가 담당할 영역과 three-body-comic 리포가 담당할 영역 분리
  - 에피소드, 장면, 패널, 결과 기록 등 핵심 객체 목록화
  - "정통 각색"이 아닌 "독자적 재해석" 전제를 운영 문서에 반영

### Phase 2: 에피소드/패널 데이터 모델 설계
- **담당**: 대구루
- **입력**: 5화 파일럿, 에피소드 매핑, 제작 단계 정의
- **출력**: episode/panel 상태 모델
- **세부**:
  - 에피소드 상태: 대본, 콘티, 생성, 리뷰, 배포, 관측
  - 패널 상태: 미생성, 생성중, 리뷰대기, 수정중, 확정
  - 작업 분배 단위: 에피소드 단위와 패널 단위의 경계 정의

### Phase 3: 이미지 생성 실행 표준화
- **담당**: 대구루
- **입력**: Codex CLI 이미지 생성 방식, OpenClaw(GLM) 사용 흐름, 이미지 생성 가이드
- **출력**: 재현 가능한 실행 절차와 프롬프트 템플릿
- **세부**:
  - OpenClaw가 Codex CLI 이미지 생성을 못 한다고 오해하는 지점 정리
  - 프롬프트 조립 규칙: 대본 -> 캐릭터 -> 구도 -> 스타일 -> 제약조건
  - 실패 시 재시도 규칙, 대체 프롬프트, A/B 선택 루프 정의

### Phase 4: 가설 검증 및 results 템플릿 설계
- **담당**: 레노버
- **입력**: H1~H6 가설, 파일럿 5화 전략, GA4 필요 데이터
- **출력**: `results.md` 템플릿과 검증 상태 기준
- **세부**:
  - 우선 검증 가설 H1, H2, H5 중심 구조 정의
  - 상태: 대기, 수집중, 검증완료
  - 관측 항목: 페이지뷰, 체류시간, 복귀율, 정성 코멘트

### Phase 5: 팀 병렬 운영 루프 및 리뷰 흐름 설계
- **담당**: 대구루
- **입력**: CLE1 카드 기반 선택, CLE2 task 구조, 팀 역할
- **출력**: 역할별 운영 규칙
- **세부**:
  - 대구루, 레노버, 사람 팀원의 작업 경계 명시
  - Discord 알림 트리거와 PR 리뷰 루프 정의
  - 작업 완료 기준과 handoff 포인트 정의

### Phase 6: CLE2 대시보드/문서 통합
- **담당**: 대구루
- **입력**: Phase 1-5 결과
- **출력**: CLE2-7 대시보드 반영, task 문서 정합성 확보
- **세부**:
  - app.js TASKS_DATA에 CLE2-7 반영
  - task 문서와 대시보드 상태 동기화
  - 후속 구현 이슈 분리 기준 정의

## 의존성
- `three-body-comic` PRD 및 이미지 생성 가이드
- `creative-loop-engineering`의 CLE1 설계
- Codex CLI 이미지 생성 실행 가능 환경

## 리스크
- 이미지 생성 품질 문제를 도구 사용법 문제와 모델 한계 문제로 구분하지 못하면 해결이 흐려질 수 있음
- 브라우저 프런트엔드에서 GitHub 쓰기 권한을 다루는 구조는 보안상 취약하므로 장기적으로는 서버 또는 GitHub App 전환이 필요함
- GA4는 설계와 실제 운영 연결 사이에 권한/계정 변수가 존재함
