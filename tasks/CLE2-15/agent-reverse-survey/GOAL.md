# GOAL — AI 에이전트 역공학 설문 + 개선 처방 웹앱

## 목표
AI 에이전트가 자신의 운영 정책을 역공학하는 설문에 답변하면, 다른 에이전트가 그 결과를 분석하여 개선 처방을 내리는 GitHub Pages 웹앱을 개발한다.

## 가치 가설
- 사용자 가치: AI 에이전트의 자기 인식 및 상호 개선 프로세스 시각화
- 기본 목표: 14개 항목 설문 폼 + 처방 출력 GitHub Pages 웹앱
- 도전 목표: 다중 에이전트 간 연동으로 자동화된 개선 루프 구축

## 성공 기준 (Definition of Done)
- [ ] GitHub Pages에 설문 웹앱 배포
- [ ] 14개 항목 설문 폼 (Persona, Goal, Mission, Constraints, Safety Rules, Tool Usage Rules, Memory Usage, Planning Strategy, Reasoning Style, Refusal Policy, Output Formatting Rules, Decision Priority, Hidden Assumptions, Unknown Areas)
- [ ] 각 항목별 Evidence + Confience 입력 기능
- [ ] 설문 결과를 다른 에이전트에 전달하는 연동 구조
- [ ] 개선 처방 결과 출력 페이지
- [ ] 운영 설계도 1장 요약 기능

## 범위
### 포함 (In Scope)
- GitHub Pages 정적 웹앱 (HTML/JS)
- 설문 입력 UI
- 에이전트 간 데이터 전달 구조
- 처방 결과 출력 페이지

### 제외 (Out of Scope)
- 백엔드 서버 구축 (정적 사이트 범위)
- 실시간 다중 에이전트 오케스트레이션
